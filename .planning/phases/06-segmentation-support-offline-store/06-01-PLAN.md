<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/user/segment | Authenticated user writes their own segment preference |
| client → /catalog?segment= | Untrusted query param influences server-side Payload query |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | POST /api/user/segment | mitigate | Zod enum validation on segment value using SEGMENT_VALUES tuple; reject unknown strings with 400 |
| T-06-02 | Elevation | POST /api/user/segment | mitigate | auth() session required; update scoped to session.user.id only — no userId param accepted from body |
| T-06-03 | Tampering | /catalog ?segment= | mitigate | isSegment() type guard before mapping to categories; unknown values silently ignored (no SQL injection surface since Payload Local API) |
| T-06-04 | Information Disclosure | localStorage segment value | accept | Segment is a non-sensitive personalization preference; no PII; acceptable to store client-side |
</threat_model>
