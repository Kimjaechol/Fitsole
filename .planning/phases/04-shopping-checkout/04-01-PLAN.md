2. Create `src/lib/cart/store.ts` — Zustand store per D-01 with localStorage persistence. Follow the exact same `create(persist(...))` pattern from scan store. Actions:
   - `addItem(item: Omit<CartItem, 'id'>)` — generates UUID id, pushes to items array. If same productId+size+designId exists, increment quantity instead.
   - `removeItem(id: string)` — filter out by cart item id
   - `updateQuantity(id: string, quantity: number)` — update quantity, remove if <= 0
   - `clearCart()` — empty all items
   - `totalPrice()` — computed: sum of (price + bundleInsolePrice if includesInsole) * quantity for each item
   - `itemCount()` — computed: sum of quantities
   Persist key: `fitsole-cart`. Partialize: persist entire items array (small data).

```typescript
export const orderStatusEnum = pgEnum("order_status", [
  "pending",         // 주문 대기
  "paid",            // 결제 완료
  "designing",       // 인솔 설계중
  "manufacturing",   // 제작중
  "shipping",        // 배송중
  "delivered",       // 배송 완료
  "cancelled",       // 취소됨
]);

export const orders = pgTable("orders", {
  id: uuid("id").defaultRandom().primaryKey(),
  userId: uuid("user_id").notNull().references(() => users.id, { onDelete: "cascade" }),
  orderNumber: text("order_number").notNull().unique(), // FS-YYYYMMDD-XXXX format
  status: orderStatusEnum("status").default("pending").notNull(),
  totalAmount: real("total_amount").notNull(),          // KRW
  shippingName: text("shipping_name").notNull(),
  shippingPhone: text("shipping_phone").notNull(),
  shippingZipcode: text("shipping_zipcode").notNull(),
  shippingAddress: text("shipping_address").notNull(),
  shippingDetailAddress: text("shipping_detail_address"),
  paymentKey: text("payment_key"),                      // Toss Payments paymentKey
  paymentMethod: text("payment_method"),                // card, kakaopay, naverpay, tosspay, transfer
  paidAt: timestamp("paid_at"),
  createdAt: timestamp("created_at").defaultNow().notNull(),
  updatedAt: timestamp("updated_at").defaultNow().notNull(),
});

export const orderItems = pgTable("order_items", {
  id: uuid("id").defaultRandom().primaryKey(),
  orderId: uuid("order_id").notNull().references(() => orders.id, { onDelete: "cascade" }),
  productId: text("product_id").notNull(),              // Payload product ID (text, not FK)
  productName: text("product_name").notNull(),
  size: real("size").notNull(),                         // mm
  price: real("price").notNull(),                       // shoe price KRW
  bundleInsolePrice: real("bundle_insole_price"),       // insole add-on KRW
  includesInsole: text("includes_insole").default("false").notNull(), // boolean as text
  designId: uuid("design_id").references(() => insoleDesigns.id, { onDelete: "set null" }),
  designSource: text("design_source"),                  // 'general' | 'professional'
  quantity: real("quantity").notNull(),
  createdAt: timestamp("created_at").defaultNow().notNull(),
});
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client cart → server | Cart data is client-side; server must validate prices at checkout, not trust client prices |
| localStorage | Cart in localStorage can be tampered; treat as untrusted input at checkout |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | T (Tampering) | Cart prices in localStorage | mitigate | Server-side price lookup from Payload at checkout time — never trust client-sent prices (Plan 02/03) |
| T-04-02 | I (Info Disclosure) | Cart contents | accept | Cart is client-only localStorage, no PII, low risk |
</threat_model>
