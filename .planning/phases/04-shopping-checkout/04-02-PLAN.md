<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client cart -> checkout API | Cart prices from localStorage — untrusted |
| Toss SDK -> our success URL | Payment result needs server-side verification |
| Daum API script | Third-party script loaded dynamically |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03 | T (Tampering) | /api/checkout/prepare | mitigate | Server fetches product prices from Payload DB, ignores client-sent prices |
| T-04-04 | S (Spoofing) | checkout/prepare | mitigate | Auth session check — reject 401 if not logged in |
| T-04-05 | T (Tampering) | Toss success callback | mitigate | Plan 03 webhook verifies payment server-side via Toss API (not trusting redirect params alone) |
| T-04-06 | I (Info Disclosure) | Daum Postcode script | accept | Public API, only returns address data, no PII sent to Daum |
</threat_model>
