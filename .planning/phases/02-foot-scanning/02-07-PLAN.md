**Create `src/app/api/scan/upload/route.ts`** per RESEARCH Open Question 1:
- POST endpoint for receiving video upload
- Check auth session
- Accept multipart form data with video file and scanId
- Per RESEARCH: upload directly to Python backend (Vercel has 4.5MB limit, incompatible with large video)
- Proxy the upload to Python backend at configurable URL (env var MEASUREMENT_SERVICE_URL, default 'http://localhost:8000')
- Forward video file as multipart to Python backend POST /scan/process
- Alternative path: if using S3/R2 presigned upload, generate presigned URL and return to client
- Update footScans record status to 'processing' after successful upload
- Return { scanId, status: 'processing' }
- Error handling: return 500 with Korean message "영상 업로드에 실패했습니다" per UI-SPEC

**Create `src/app/api/scan/process/route.ts`:**
- POST endpoint to trigger processing and save results
- Check auth session
- Accept { scanId, footSide }
- Call Python backend endpoints sequentially:
  1. POST {MEASUREMENT_SERVICE_URL}/scan/process -> foot scan results
  2. POST {MEASUREMENT_SERVICE_URL}/gait/analyze -> gait analysis results
  3. POST {MEASUREMENT_SERVICE_URL}/pressure/estimate -> pressure estimation results
- Save all results to DB tables (footMeasurements, gaitAnalysis, pressureDistribution)
- Update footScans status to 'completed', set completedAt
- If any step fails: update footScans status to 'failed', set errorMessage
- Update footScans.processingStage at each step for status polling
- Return { scanId, status: 'completed' | 'failed' }

- Read existing component first
- Replace empty state placeholder with actual scan data display
- Fetch user's scan history from /api/scan/history (or direct DB query if server component)
- If scans exist: show list of scan results with:
  - Date (Korean format)
  - Quality badge (우수/보통/부족)
  - Foot side (왼발/오른발)
  - Key measurements summary (foot length, ball width)
  - Link to full results (/scan/results/[id])
- If no scans: keep existing EmptyState with updated CTA linking to /scan
- "새로 측정하기" button at top
