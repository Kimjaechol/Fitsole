Create all 7 pipeline modules per RESEARCH.md architecture:

**`frame_extractor.py`** per D-01:
- Function `extract_frames(video_path: Path, output_dir: Path, fps: float = 2.0) -> list[Path]`
- Use cv2.VideoCapture to read video
- Extract frames at 2 FPS (every 0.5 seconds per RESEARCH discretion decision)
- Save frames as JPEG to output_dir with sequential naming (frame_0001.jpg, frame_0002.jpg, ...)
- Return list of frame file paths
- Log frame count extracted

**`quality_filter.py`** per D-19, SCAN-03, SCAN-06:
- Function `check_frame_quality(frame_path: Path) -> dict` returning {blur: bool, dark: bool, score: float}
- Blur detection: cv2.Laplacian variance. Threshold: < 100 = blurry (per RESEARCH "don't hand-roll" section)
- Darkness detection: average pixel luminance < 50 = too dark
- Score: 0-100 based on sharpness + brightness combination
- Function `filter_frames(frame_paths: list[Path], min_score: float = 40.0) -> tuple[list[Path], list[dict]]`
- Returns (good_frames, quality_reports)
- Function `compute_scan_quality(quality_reports: list[dict]) -> tuple[float, str]`
- Returns (score 0-100, label 'good'|'fair'|'poor')
- Thresholds: >= 70 = 'good', >= 40 = 'fair', < 40 = 'poor'

**`a4_detector.py`** per D-02, SCAN-02:
- Function `detect_a4_paper(frame: np.ndarray) -> tuple[np.ndarray, float] | None`
- Per RESEARCH Pattern 2: grayscale -> GaussianBlur -> Canny edges -> findContours -> filter by 4 corners + A4 aspect ratio (1.3-1.55 tolerance)
- Return (corner_points, pixels_per_mm) or None if not detected
- Function `calibrate_from_frames(frame_paths: list[Path]) -> float | None`
- Try A4 detection on first 5 frames, return median pixels_per_mm
- Return None if A4 not detected (triggers re-scan prompt per D-19)

**`sfm_reconstructor.py`** per D-01, D-03, D-04, SCAN-04:
- Function `run_sfm_pipeline(frames_dir: Path, output_dir: Path) -> pycolmap.Reconstruction | None`
- Per RESEARCH Pattern 1: feature extraction (SIFT per D-04) -> sequential matching (NOT exhaustive per Pitfall 2) -> incremental mapping
- Use pycolmap.extract_features with SiftExtractionOptions
- Use pycolmap.match_sequential with SequentialMatchingOptions (video frames have natural overlap)
- Use pycolmap.incremental_mapping
- Return first reconstruction or None on failure
- Function `get_point_cloud(reconstruction: pycolmap.Reconstruction) -> np.ndarray`
- Extract 3D points as numpy array from reconstruction

**`mesh_generator.py`** per RESEARCH:
- Function `point_cloud_to_mesh(points: np.ndarray, colors: np.ndarray | None = None) -> open3d.geometry.TriangleMesh`
- Decimate point cloud to ~50K points if larger (per Pitfall 4)
- Estimate normals with Open3D
- Run Poisson surface reconstruction
- Simplify mesh to ~20K faces (per Pitfall 4, target GLB < 5MB)
- Return mesh with vertex colors if available

**`measurement_extractor.py`** per D-11, SCAN-04:
- Function `extract_measurements(mesh, pixels_per_mm: float) -> dict`
- Extract from calibrated mesh: foot_length (max Z extent), ball_width (max X extent at widest point), instep_height (max Y at instep region), arch_height (min Y in arch region), heel_width (X extent at heel), toe_length (Z extent of toe region)
- All values in mm using pixels_per_mm calibration factor
- Return dict with all 6 measurement keys matching FootMeasurement type from types.ts

**`model_exporter.py`** per RESEARCH discretion (GLB format):
- Function `export_to_glb(mesh, output_path: Path) -> Path`
- Convert Open3D mesh to trimesh
- Export as GLB binary format
- Return output path
- Target file size < 5MB per Pitfall 4
