## Visual Hierarchy

**Primary focal point:** The camera viewfinder dominates during scanning steps (fullscreen). The recording button (72px, accent fill, bottom-center) is the single interactive element during video capture. After scanning, the 3D foot model with measurement overlay becomes the primary focal point on the results page.

**Hierarchy order:**
1. Camera viewfinder + recording button -- fullscreen immersive, accent-colored recording control
2. Real-time guidance overlay -- white text on semi-transparent dark bg, positioned top-center
3. Step progress indicator -- small dots at top of screen, current step accent-filled
4. Results: 3D foot model -- largest visual element on results page (360px height)
5. Results: measurement values -- body text in card containers alongside the model
6. Results: pressure heatmap -- color-coded overlay on 3D model, toggle-able
7. Results: bottom CTA -- accent fill, full width, lowest scroll position

## Interaction Patterns

### Camera Viewfinder
- Fullscreen immersive mode: hide bottom tab bar, top nav, and status bar (if possible via Fullscreen API)
- Dark overlay (rgba(0,0,0,0.5)) outside the scan area to focus attention
- A4 paper detection: green border highlight on detected A4 rectangle, red border if not detected
- Real-time frame quality: blur detection shows amber warning text at top, lighting warning shows amber text
- Back arrow (top-left, 44px) exits scanning flow with confirmation dialog

### Recording Button
- Idle state: 72px circle, accent fill, white camera icon (Lucide `Video`)
- Recording state: 72px circle, destructive red fill, white square stop icon, pulses with scale(1.0 -> 1.05) at 1s interval
- Tap to start: haptic feedback (navigator.vibrate(50) if available), 3-second countdown overlay before recording begins
- Tap to stop: recording ends, brief preview of captured video (2s auto-play loop)
- Disabled state: 50% opacity during processing/uploading

### Circular Motion Guide
- SVG circle (280px, dashed stroke, accent color at 30% opacity) on viewfinder
- Animated dot travels along the circle path showing suggested camera trajectory
- Progress arc fills in as user completes circular motion (0% to 100%)
- Arc color transitions: accent at 30% opacity -> solid accent as progress increases

### Walking Video Guide
- Instruction screen before recording: illustration of phone placed on floor at ground level
- 3-second countdown timer (large centered numbers, 48px, 700 weight)
- During recording: MediaPipe pose landmarks rendered as dots on video feed (accent color dots, 8px)
- Step counter displayed at top: "{n}/10 걸음" in body text on semi-transparent bg

### Step Progress Indicator
- Horizontal row of 4 dots at top of scan flow screens
- Inactive dot: 12px, slate-200 fill
- Active dot: 16px, accent fill, with subtle scale-up animation (200ms ease-out)
- Completed dot: 12px, accent-secondary (emerald) fill, with checkmark icon (8px)
- Labels below dots (label size 14px): 발 위치 | 영상 촬영 | 보행 촬영 | 결과

### Processing Screen
- Centered layout with animated illustration (not a spinner -- animated foot outline or wireframe building)
- 4 processing stages displayed as a vertical list with current stage highlighted
- Progress bar (8px height, accent fill, rounded) below the animation
- Percentage text below progress bar (body 16px)
- "약 3-5분 소요됩니다" muted text below percentage
- User can navigate away -- processing continues server-side, push notification (toast) when complete

### 3D Foot Model Viewer
- React Three Fiber canvas with OrbitControls (rotate by drag, zoom by pinch/scroll)
- Initial view: top-down 45-degree angle showing full foot
- Measurement dimension lines rendered as Three.js line segments with value labels
- Toggle button for pressure heatmap overlay (shadcn Toggle component)
- Background color: slate-50 (#F8FAFC) inside canvas for contrast with model
- Loading state: Skeleton placeholder (360px height) until model loads

### Quality Scoring
- Quality badge (QualityBadge component) appears after scan processing
- Three states with distinct colors:
  - "우수" (good): emerald-600 fill, white text
  - "보통" (fair): amber-600 fill, white text
  - "부족" (poor): red-600 fill, white text
- If "poor": RescanPrompt banner appears with specific guidance and "다시 촬영하기" CTA
- If "fair": subtle suggestion banner (amber border, amber-50 bg) but allows proceeding

### Left/Right Foot Flow
- After completing first foot scan, FootSideSelector appears
- Shows which foot was scanned (filled icon) vs remaining (outline icon)
- User taps outline foot icon to start second scan
- Results page shows both feet side by side (or tabbed on mobile)

### Scan Onboarding (First Time)
- Displayed once (tracked in localStorage: `fitsole_scan_onboarded`)
- 3-step horizontal swipeable carousel with illustrations
- Each step: illustration (placeholder Lucide icon at 80px) + title + description
- "시작하기" CTA on last step, "건너뛰기" skip link on all steps (muted text, top-right)

### Results Page
- Sticky header: "측정 결과" heading + left/right foot toggle
- 3D model section: interactive viewer with heatmap toggle
- Measurements section: grid of MeasurementCard components (2 columns on mobile, 3 on desktop)
- Each MeasurementCard: label (14px, muted) + value (24px, 700 weight) + unit (14px, muted)
- Gait analysis section: separated by heading, displays pattern/alignment/flexibility as text descriptions
- Asymmetry notice: if detected, amber Alert component between measurement sections
- Bottom CTA: full-width accent button, fixed to bottom on mobile (above bottom tab bar)

### Page Transitions
- Scan flow steps: slide-left transition (Framer Motion, 200ms ease-out)
- Results page entry: fade-in (Framer Motion, 300ms ease-out)
- Processing to results: no transition, direct navigation on completion

### Error Handling
- Camera permission denied: Dialog with settings link instruction
- Upload failure: Toast (destructive variant) with retry action
- Processing failure: Full-page error state with "다시 촬영해 주세요" CTA
- Network error: Toast (destructive variant) with auto-retry after 5 seconds

## State Management

Zustand store for scan workflow state machine:

| State | Description |
|-------|-------------|
| `idle` | Scan home, no active scan session |
| `onboarding` | First-time guide (skippable) |
| `positioning` | Step 1: A4 paper + foot placement |
| `recording_foot` | Step 2: Video recording in progress |
| `recording_gait` | Step 3: Walking video recording |
| `uploading` | Video upload in progress (percentage tracked) |
| `processing` | Server-side SfM processing (stage tracked) |
| `results` | Scan complete, viewing results |
| `error` | Error state with recovery path |

State persisted across navigation via Zustand. Processing state polled via TanStack Query (5s interval).
