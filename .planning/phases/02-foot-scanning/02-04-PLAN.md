- Return list of angle measurements per frame

**Create `services/measurement/app/gait/gait_classifier.py`** per D-07, D-12, SCAN-09:
- Function `classify_gait(ankle_angles: list[dict]) -> str`
- Analyze ankle angle patterns over gait cycle
- Classify as 'normal', 'overpronation', or 'supination'
- Overpronation: excessive inward roll (> 4 degrees from neutral)
- Supination: excessive outward roll (> 4 degrees from neutral)
- Normal: within 4 degrees of neutral

- Function `compute_pronation_supination(ankle_angles: list[dict]) -> str`
- Return 'neutral', 'pronation', or 'supination' based on mean ankle alignment

**Create `services/measurement/app/gait/arch_analyzer.py`** per D-07, D-08, D-12:
- Function `compute_arch_flex_index(landmarks_over_time: list[dict]) -> float`
- Compare arch height during stance phase vs swing phase
- Return flexibility index 0-1 (0 = rigid, 1 = very flexible)
- Use weight-bearing deformation reference data per D-08: midfoot height decreases 15.1-18.2% under load

**Create `services/measurement/app/api/gait.py`:**
- APIRouter with prefix "/gait"
- POST `/analyze` endpoint -- accepts walking video file (UploadFile), scanId (str)
- Processes: detect_landmarks -> compute_ankle_angles -> classify_gait + compute_pronation_supination + compute_arch_flex_index
- Returns: { scanId, gaitPattern, ankleAlignment, archFlexibilityIndex }

**Create `services/measurement/app/pressure/estimator.py`** per D-09, SCAN-10:
- Function `estimate_pressure(weight: float, foot_length: float, ball_width: float, arch_height: float, gender: str, age: float) -> dict`
- Per D-09: AI-estimated pressure distribution using inputs (weight, foot size, arch height, gender, age)
- Per RESEARCH Open Question 3: Start with rule-based heuristic model (known pressure distribution patterns for arch types)
- Arch type classification from arch_height:
  - Low arch (flat foot): < 15mm -> higher medial pressure
  - Normal arch: 15-30mm -> balanced pressure
  - High arch: > 30mm -> higher lateral and heel pressure
- Weight distribution: heavier weight increases forefoot and heel pressure proportionally
- Gender adjustment: female feet typically have narrower heels relative to forefoot
- Age adjustment: older adults tend to have flatter arches and wider forefeet
- Return dict with: pressureGrid (2D array 20x10 representing plantar surface), peakPressureZones (list of {x, y, intensity, label}), overallDistribution (forefoot_pct, midfoot_pct, rearfoot_pct)

**Create `services/measurement/app/pressure/heatmap_generator.py`** per D-10, D-23:
- Function `generate_heatmap(pressure_grid: list[list[float]]) -> dict`
- Takes 2D pressure grid
- Normalize to 0-1 range
- Identify high-pressure zones (> 0.7 threshold) with labels: "전족부" (forefoot), "중족부" (midfoot), "후족부" (rearfoot), "내측" (medial), "외측" (lateral)
- Return { normalizedGrid, highPressureZones: [{x, y, intensity, label}], warnings: [str] }
- Warnings for zones exceeding 0.85 threshold: "{zone_label} 부위에 과도한 압력이 감지되었습니다"

**Create `services/measurement/app/api/pressure.py`:**
- APIRouter with prefix "/pressure"
- POST `/estimate` endpoint
- Accept JSON body: { weight, footLength, ballWidth, archHeight, gender, age, scanId }
- Validate inputs with Pydantic model
- Call estimate_pressure -> generate_heatmap
- Return: { scanId, heatmapData (2D grid), highPressureZones, warnings }
