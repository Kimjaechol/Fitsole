    1. Create src/lib/validators/auth.ts -- Zod validation schemas:
       ```typescript
       import { z } from "zod";

       export const loginSchema = z.object({
         email: z.string().email("올바른 이메일 주소를 입력해 주세요."),
         password: z.string().min(8, "비밀번호는 8자 이상이어야 합니다."),
       });

       export const signUpSchema = z.object({
         email: z.string().email("올바른 이메일 주소를 입력해 주세요."),
         password: z.string().min(8, "비밀번호는 8자 이상이어야 합니다."),
         confirmPassword: z.string(),
         name: z.string().min(1, "이름을 입력해 주세요."),
       }).refine((data) => data.password === data.confirmPassword, {
         message: "비밀번호가 일치하지 않습니다.",
         path: ["confirmPassword"],
       });

       export const resetPasswordSchema = z.object({
         email: z.string().email("올바른 이메일 주소를 입력해 주세요."),
       });
       ```
       NOTE: If Zod v4 has breaking API changes from v3, check `z.object` and `.refine()` still work. If not, adapt to Zod v4 API.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Signup API | Untrusted user input (email, password) crosses to server |
| Client -> NextAuth API | Untrusted credentials for authentication |
| Server -> Neon DB | Server trusts database, connection via TLS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | POST /api/auth/signup | mitigate | Zod validation on all inputs; bcryptjs hash with cost 10; unique email constraint |
| T-01-02 | Spoofing | Credentials authorize() | mitigate | bcrypt.compare for password verification; generic error message "이메일 또는 비밀번호가 올바르지 않습니다" prevents user enumeration |
| T-01-03 | Information Disclosure | JWT token | mitigate | httpOnly cookies via Auth.js default; JWT contains only user id, not password |
| T-01-04 | Tampering | Auth forms | mitigate | Auth.js built-in CSRF protection |
| T-01-05 | Elevation of Privilege | /mypage routes | mitigate | middleware.ts enforces auth on protected routes |
| T-01-06 | Denial of Service | POST /api/auth/signup | accept | No rate limiting in v1; low-value target pre-launch. Add rate limiting if abuse detected. |
</threat_model>
