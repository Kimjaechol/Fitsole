## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Reset Password API | Email address input for password reset |
| Reset Password API -> Resend | Outbound email with reset token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | POST /api/auth/reset-password | mitigate | Always return 200 regardless of email existence; prevents user enumeration |
| T-03-02 | Spoofing | Reset token | mitigate | crypto.randomUUID() for unpredictable tokens; 1-hour expiry in verificationTokens table |
| T-03-03 | Tampering | Auth forms | mitigate | Zod validation on all inputs; react-hook-form prevents raw submission |
| T-03-04 | Denial of Service | Password reset emails | accept | No rate limiting on reset endpoint in v1; Resend has built-in rate limits |
</threat_model>
