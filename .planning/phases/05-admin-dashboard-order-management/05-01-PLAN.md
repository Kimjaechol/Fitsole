   - `ORDER_STATUS_LABELS` map: { pending: "주문 대기", paid: "주문확인", designing: "인솔설계", manufacturing: "제작중", shipping: "배송중", delivered: "완료", cancelled: "취소됨" } (per D-01)

   - `OrderDetail` interface (for detail page): full order fields + items array with productName/size/price/includesInsole/designSource + insole design summary (designParams, lineType) per D-03

3. Create `GET /api/orders` route:
   - Auth check with `auth()`, return 401 if no session
   - Query orders table filtered by `userId = session.user.id` (IDOR prevention)
   - Join with orderItems to get itemCount and firstItemName
   - Sort by createdAt desc
   - Return `OrderSummary[]`

4. Create `GET /api/orders/[id]` route:
   - Auth check, return 401 if no session
   - Query order by id AND userId (IDOR prevention)
   - Join orderItems, and for items with designId, join insoleDesigns to get designParams and lineType
   - Return `OrderDetail` with all fields per D-03
   - Return 404 if order not found or belongs to different user
  </action>
