# FitSole 도매 파트너 신발 등록 매뉴얼

> **개정일:** 2026-04-11
> **대상 독자:** FitSole와 신발 공급 계약을 체결한 **도매·브랜드 파트너** 담당자
> **문의 채널:** `partners@fitsole.kr` (플레이스홀더), 파트너 포털 상의 "파트너 지원" 메뉴
> **관련 문서:** 내부 운영 SOP는 [SOP-shoe-interior-scan-merge.md](./SOP-shoe-interior-scan-merge.md), 연구 배경은 [docs/research/01-shoe-interior-scanning.md](../research/01-shoe-interior-scanning.md) 및 [docs/research/02-shoe-size-grading.md](../research/02-shoe-size-grading.md) 참조

## 개요

FitSole는 소비자가 스마트폰으로 자신의 발을 측정하면, 과학적 분석을 통해 **개인 맞춤 인솔을 설계하고 신발과 세트로 판매**하는 커스텀 신발 이커머스 플랫폼이다. 소비자는 "이 신발이 내 발에 맞을까?"를 고민할 필요 없이, 자신의 발 측정값과 가장 잘 매칭되는 신발 모델을 추천받고, 동일 주문 안에서 해당 신발 전용으로 설계된 인솔을 함께 받게 된다.

이 모델이 작동하려면 FitSole는 카탈로그의 모든 신발 모델에 대해 **"내부에서 실제로 얼마나 공간이 있는가"**를 정량적으로 알고 있어야 한다. 동일한 "270mm" 표기라도 브랜드와 모델에 따라 내부 길이, 토박스 부피, 힐컵 깊이 등이 최대 **11 mm 수준까지 차이**가 날 수 있다는 점이 연구 문서(`docs/research/01-shoe-interior-scanning.md`)에서 확인되었다. 이 편차는 FitSole의 인솔 설계 입력값에 직접 영향을 주기 때문에, 파트너가 공급하는 모든 신발 모델은 FitSole 측정팀의 스캔 + 병합 파이프라인을 거친 뒤에만 카탈로그에 등록될 수 있다.

파트너가 이 절차를 수행함으로써 얻는 가치는 다음과 같다.

- **반품률 감소:** 소비자의 실제 발 치수와 신발 내부 치수가 매칭되기 때문에 사이즈 미스핏으로 인한 반품이 구조적으로 줄어든다.
- **타겟팅된 고객 세그먼트 노출:** FitSole는 발 건강 고민자, 일반 소비자, 운동선수 등 카테고리별로 고객을 분류하여 추천하므로, 모델이 정확히 어떤 세그먼트에 노출될지 데이터로 확인할 수 있다.
- **맞춤 인솔을 통한 프리미엄화:** 일반 유통 채널에서는 표현하기 어려운 "내 발에 맞춰 설계된 인솔 포함" 소구점을 파트너 브랜드에 그대로 부여할 수 있다.
- **원스톱 제조·배송 연계:** FitSole가 인솔을 자체 설계하고 외부 제조 공장과 연동하여 배송까지 처리하므로, 파트너는 신발 단품 공급에만 집중할 수 있다.

## FitSole이 제공하는 데이터

각 신발 모델이 FitSole 카탈로그에 등록되면 파트너는 다음 데이터를 파트너 포털 또는 보고서 형식으로 받게 된다.

- **스캔된 내부 치수** — 앵커 사이즈 기준 실측값
  - 내부 길이 (`internalLength`)
  - 내부 너비 (`internalWidth`)
  - 힐컵 깊이 (`heelCupDepth`)
  - 아치 지지점 X (`archSupportX`)
  - 토박스 부피 (`toeBoxVolume`)
  - 발등 여유 (`instepClearance`)
- **사이즈별 그레이딩 예측값** — 한국 사이즈 **230–290 mm** 범위에 대해 각 치수의 예측값을 5 mm 간격으로 제공
- **그레이딩 검증 경고(validation_warnings)** — 어떤 사이즈 영역에서 예측 불확실성이 큰지에 대한 의미 요약
- **품질 지표** — `alignment_rmse` (mm)와 `overlap_percentage` (%), 그리고 "discrepancy_count"(스캔 간 치수 불일치 수)
- **카탈로그 요약 리포트** — 모델 썸네일, 추천 대상 세그먼트, 기본 사이즈, 가격, 재고 상태

이 데이터는 파트너의 내부 시스템(ERP, 상품 DB)과의 연동을 전제로 하지 않는다. 우선 이메일/파트너 포털 다운로드로 제공되며, API 연동은 분기별 로드맵에 따라 확장된다.

## 도매 파트너 워크플로우

1. **신발 모델 등록 요청서 제출** — "신발 모델 등록 양식"의 모든 필수 항목을 작성하여 `partners@fitsole.kr` 로 전송하거나 파트너 포털에 업로드한다.
2. **앵커 사이즈 샘플 발송** — FitSole 측정팀이 요청한 앵커 사이즈(기본 270 mm)의 물리적 샘플을 FitSole 측정 센터로 발송한다. 다중 앵커 사이즈를 함께 제공하면 그레이딩 정확도가 향상된다.
3. **FitSole 측정팀 스캔 + 병합** — 측정팀이 Revopoint 광학 스캐너와 알지네이트 캐스트를 결합한 하이브리드 절차로 내부 치수를 획득하고, 관리자 대시보드에서 병합(`/api/admin/shoe-scan/merge`)을 수행한다. 내부 작업 절차는 [SOP-shoe-interior-scan-merge.md](./SOP-shoe-interior-scan-merge.md) 에 명시되어 있다.
4. **치수/그레이딩 검수 및 승인** — FitSole는 병합 결과와 그레이딩 예측값을 담은 승인 요청을 파트너에게 전달한다. 파트너는 "사이즈 그레이딩 결과 이해하기" 섹션의 체크리스트에 따라 승인 또는 재스캔을 요청할 수 있다.
5. **카탈로그 등록, 판매 개시** — 승인 후 모델이 FitSole 카탈로그에 등재되고, 추천 알고리즘의 후보군에 진입한다.
6. **주문 발생 시 알림 → 신발 + 인솔 배송 흐름** — 소비자 주문이 발생하면 파트너에게 알림이 전달되고, 파트너는 계약된 리드타임 내에 신발을 FitSole 또는 최종 고객에게 발송한다. 인솔은 FitSole가 설계·제작하여 세트로 포장·배송된다.

| 단계 | 담당        | 일반 소요         | 산출물                                    |
| ---- | ----------- | ----------------- | ----------------------------------------- |
| 1    | 파트너      | 즉시              | 등록 요청서                               |
| 2    | 파트너      | 3 영업일 이내     | 앵커 샘플 택배 발송                       |
| 3    | FitSole     | 수령 후 1 영업일  | 병합 결과 JSON                            |
| 4    | 공동        | 2 영업일          | 승인 응답                                 |
| 5    | FitSole     | 1 영업일          | 카탈로그 노출 시작                        |
| 6    | 공동        | 상시              | 주문 → 배송                               |

## 앵커 사이즈 선정 기준

**권장:** 중간 사이즈 1 개 (`270 mm`) + 가능하면 작은 사이즈(`250 mm`)와 큰 사이즈(`290 mm`) 각 1 개씩 추가, 총 3 개의 앵커.

**이유:** FitSole의 그레이딩 API는 다중 앵커가 제공될 경우 **구간별 선형 보간(piecewise linear interpolation)** 을 사용하여 각 사이즈 구간에서 독립적으로 편향을 줄인다. 단일 앵커만 제공될 경우 모든 사이즈를 하나의 선형 추세로 외삽하게 되어 양 끝(예: 230 mm, 290 mm) 구간의 오차가 커질 수 있다. 세부 알고리즘과 편차 데이터는 `docs/research/02-shoe-size-grading.md` 에 정리되어 있다.

**실무 팁:**

- 2 개 이상 앵커를 제공하면 `/api/admin/shoe-scan/grade` 응답에서 `used_piecewise: true` 를 확인할 수 있고, `validation_warnings` 빈도가 낮아지는 것을 관찰할 수 있다.
- 앵커 사이즈는 **표본 중 평균적인 제조 편차 범위** 를 가진 것을 선정한다. 특이 공정(예: 특수 보강)이 적용된 샘플은 피한다.
- 좌·우 페어 중 한 쪽(보통 오른발)을 기준으로 일관되게 제공한다. 좌우 비대칭 모델은 사전에 공지한다.

## 스캔 의뢰 절차

FitSole는 두 가지 스캔 의뢰 옵션을 제공한다.

### 옵션 A — FitSole 측정팀에 샘플 발송 (권장)

- **수령 주소:** FitSole 측정 센터 (파트너 포털 계약 정보란 참조)
- **회수 여부:** 측정 완료 후 24시간 이내 회수 배송 가능. 회수 배송료는 파트너 부담이 기본값이나 월 10 모델 이상 공급 시 FitSole 부담으로 전환.
- **소요 기간:** 도착 후 1 영업일 이내 병합 결과 전달

### 옵션 B — 파트너 자체 장비로 스캔

- **허용 파일 포맷:** `.stl` / `.obj` / `.ply` / `.gltf` / `.glb` (파일당 최대 100 MB)
- **파일명 규칙:** `{파트너ID}-{모델슬러그}-{사이즈}.stl` 예) `nike-airforce1-low-270.stl`
- **업로드 경로:** SFTP 서버 또는 이메일 첨부(현재는 플레이스홀더, 파트너별 전용 업로드 URL 제공 예정)
- **품질 요구사항:** 내부 표면 커버리지 ≥ 60 %, alignment 기준 마커 포함, 각 치수 특징점 관찰 가능

> 내부 운영 세부(알지네이트 혼합 비율, 경화 시간, Revopoint 모드 등)는 파트너에게 공개되지 않으며, FitSole 내부 운영 SOP ([SOP-shoe-interior-scan-merge.md](./SOP-shoe-interior-scan-merge.md)) 에 관리된다.

## 사이즈 그레이딩 결과 이해하기

병합 후 FitSole가 파트너에게 공유하는 그레이딩 응답은 다음과 같은 JSON 형식이다.

```json
{
  "anchors_used": 2,
  "predictions": {
    "230": { "internalLength": 241.2, "internalWidth": 92.1, "heelCupDepth": 34.8 },
    "235": { "internalLength": 246.5, "internalWidth": 93.0, "heelCupDepth": 35.2 },
    "240": { "internalLength": 251.9, "internalWidth": 93.9, "heelCupDepth": 35.6 },
    "270": { "internalLength": 283.2, "internalWidth": 99.3, "heelCupDepth": 38.0 },
    "290": { "internalLength": 303.8, "internalWidth": 102.7, "heelCupDepth": 39.5 }
  },
  "validation_warnings": [
    "extrapolation_beyond_smallest_anchor: 230–245 mm 구간은 최소 앵커(250 mm) 외삽 결과입니다",
    "width_variance_high: internalWidth 예측 분산이 ±2.0 mm 입니다"
  ],
  "used_piecewise": true
}
```

각 필드의 의미:

- **anchors_used** — 그레이딩에 사용된 앵커 사이즈의 개수. 숫자가 클수록 예측 정확도 향상.
- **predictions** — 한국 사이즈(mm) 별 예측 내부 치수. 키는 문자열 형태의 사이즈(`"230"`, `"235"` …).
- **validation_warnings** — 예측 품질에 대한 경고. 다음 3가지가 대표적이다:
  1. `extrapolation_beyond_smallest_anchor` / `extrapolation_beyond_largest_anchor` — 앵커 범위 밖 사이즈는 외삽이므로 추가 샘플을 권장
  2. `width_variance_high` — 너비 편차가 커서 소비자 핏 경험이 사이즈별로 일관되지 않을 가능성
  3. `anchor_inconsistency` — 여러 앵커 간 트렌드가 비선형이어서 앵커 중 하나를 재측정 권장
- **used_piecewise** — `true` 면 다중 앵커 사이 구간별 선형 보간이 사용되었음을 의미. `false` 면 단일 앵커 기반 외삽이며, 추가 앵커 제공 시 품질 개선 가능.

### 파트너 승인 체크리스트

카탈로그 등재 전, 파트너는 다음 항목을 검토하여 승인 여부를 결정한다.

- [ ] `alignment_rmse` < 1.0 mm (품질 지표에서 확인)
- [ ] `overlap_percentage` > 70 % (품질 지표에서 확인)
- [ ] 주요 타겟 사이즈(250 – 280 mm) 구간에 `validation_warnings` 가 없음
- [ ] `used_piecewise: true` 또는, 단일 앵커인 경우 타겟 범위가 앵커 ±20 mm 이내
- [ ] `predictions` 의 `internalLength` 가 사이즈별로 **단조 증가** (역전되는 구간이 없음)
- [ ] 샘플의 특이 공정 여부 공지가 반영되어 있음

승인에 문제가 있을 경우 파트너는 재스캔 요청 또는 추가 앵커 샘플 발송을 선택할 수 있다.

## 재고 및 주문 연동

- **주문 알림:** 소비자 주문이 확정되면 파트너 포털과 담당자 이메일로 알림이 전송된다. 파트너는 확정 후 리드타임 내에 발송 상태를 업데이트한다.
- **신발 배송 흐름:** 기본값은 "파트너 → FitSole 창고 → 인솔 합포장 → 최종 고객" 흐름이다. 대형 파트너의 경우 "파트너 → 최종 고객 직배송 + FitSole 인솔 별도 배송" 옵션을 선택할 수 있다(합포장 여부는 월간 볼륨에 따라 재협의).
- **재고 업데이트 주기:** 기본 주 1 회 CSV 업로드(파트너 포털). 실시간 API 연동은 로드맵 Phase 후반에 제공 예정. 품절 시 즉시 이메일 또는 포털에서 "재고 없음" 처리한다.
- **품절 처리:** 주문 후 품절 판명 시 24시간 이내 FitSole 운영팀에 알린다. FitSole는 소비자에게 자동 환불 또는 대체 모델을 제안한다.
- **반품 회수:** 반품은 기본적으로 FitSole 창고에서 수거하며, 파트너에게는 반품 로그만 공유된다(파트너 부담 여부는 반품 사유에 따름, 아래 QA 정책 참조).

## 품질 관리 및 반품 정책

- **정확도 기준:** 카탈로그 등재된 모델의 내부 치수가 실측과 체계적으로 어긋나는 것이 확인될 경우(예: 같은 모델에 대해 소비자 불만이 월 5건 이상), FitSole는 재스캔을 요청할 수 있다.
- **반품 수용 조건:**
  - 사이즈 미스핏(FitSole 측정값 오류): FitSole 부담
  - 제조 결함/불량: 파트너 부담, 교환 또는 환불
  - 단순 변심: 소비자 부담(FitSole 기본 정책에 따름)
- **클레임 처리 흐름:** 소비자 클레임 접수 → FitSole 운영팀 1차 검토 → 제조 결함 의심 시 파트너에게 사진/영상 전달 → 파트너 응답 2 영업일 이내 → 처리 결과 소비자 통보.
- **리콜:** 브랜드 리콜 발생 시 파트너는 즉시 FitSole에 통보하며, FitSole는 해당 모델의 판매를 즉시 중단하고 기존 주문을 운영팀이 개별 대응한다.

## 지원 채널 및 연락처

- **파트너 이메일:** `partners@fitsole.kr` (플레이스홀더 — 계약 시 전용 메일로 교체)
- **대응 시간:** 평일 10:00 – 18:00 KST, 주문 긴급 건은 포털의 "긴급 요청" 기능 사용
- **기술 지원 범위:** 등록 양식 작성, 앵커 사이즈 선정, 병합 결과 해석, 그레이딩 검증 경고 해석, 재고 연동 포맷
- **에스컬레이션:** 기술 지원에서 48시간 이내 해결되지 않는 경우 `partner-leads@fitsole.kr` 로 에스컬레이션

## 용어집

- **alignment_rmse** — Revopoint 스캔과 알지네이트 캐스트 스캔을 정합(ICP)했을 때 두 메시 간 남는 평균 제곱근 오차. 값이 작을수록 병합 품질이 좋다.
- **overlap_percentage** — 두 스캔이 서로 겹치는 표면 비율. 값이 클수록 병합 시 서로를 검증해 줄 데이터가 많다.
- **discrepancy_count** — 같은 치수 항목에 대해 두 스캔이 허용 오차를 초과하여 일치하지 않은 항목의 수. 0 이 이상적.
- **anchor (앵커 사이즈)** — 그레이딩 API가 기준으로 삼는 실측 사이즈의 내부 치수 데이터. 앵커가 많을수록 예측 정확도 향상.
- **piecewise (구간별 보간)** — 앵커가 2개 이상일 때 사이즈 구간마다 독립된 선형 회귀를 적용하는 그레이딩 방식. `used_piecewise: true` 일 때 활성.
- **fitType** — 신발의 핏 경향을 나타내는 라벨 (`narrow` / `standard` / `wide`). 파트너가 사전 표기하지 않는 경우 FitSole가 스캔 후 결정한다.
- **internalLength / internalWidth** — 각각 신발 내부 세로 길이와 가로 너비 (mm).
- **heelCupDepth** — 힐컵 뒷벽 기준 세로 깊이 (mm).
- **archSupportX** — 아치가 지지되는 중심점의 X 좌표 (mm). 인솔 설계 시 중요한 기준점.
- **toeBoxVolume** — 토박스 내부 부피. 값의 단위는 서버 응답 그대로 표기 (일반적으로 mm³).
- **instepClearance** — 발등 위 여유 공간 (mm). 값이 작을수록 발등이 꽉 조이는 핏.
